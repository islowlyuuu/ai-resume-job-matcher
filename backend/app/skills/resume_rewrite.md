# Resume Rewrite Skill

目标：在不编造经历的前提下，把简历内容改写得更贴合目标岗位。

改写原则：

- 只优化表达、顺序和强调重点，不新增不存在的公司、学历、证书、业绩数字或项目成果。
- 每条 bullet 尽量包含：业务场景 + 关键动作 + 使用技术 / 方法 + 结果影响。
- 优先突出岗位 JD 中反复出现的技术栈、业务场景和能力要求。
- 避免空泛词，例如“熟悉各种技术”“负责很多模块”“显著提升效率”，除非简历中有具体证据。
- 对经验较少的候选人，可以强调项目完整度、工程实践、问题定位和可维护性。

输出关注：

- `optimized_headline`：直接对齐目标岗位方向。
- `optimized_summary`：用 2-4 句概括候选人与岗位最相关的能力。
- `rewritten_bullets`：输出 3-6 条适合放入简历的经历 bullet。
- `edit_notes`：解释为什么这样改写更贴合岗位。
